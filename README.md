# Dillinger
## _The Last Markdown Editor, Ever_

[![N|Solid](https://cldup.com/dTxpPi9lDf.thumb.png)](https://nodesource.com/products/nsolid)

[![Build Status](https://travis-ci.org/joemccann/dillinger.svg?branch=master)](https://travis-ci.org/joemccann/dillinger)

Dillinger is a cloud-enabled, mobile-ready, offline-storage compatible,
ReactJS for Compoenent SSR.

- Type some Markdown on the left
- See HTML in the right
- ✨Magic ✨

## Features

- Fitur autentikasi registrasi dengan penggunaan database portgreSQL
- Menghitung jumlah Angsuran Perbulan menggunakan perhitungan yang berlaku

Project yang diselesaikan memiliki beberapa komunikasi setiap server dan klien saling terhungung
As [Ihrat Pramudya] writes on the [Markdown site][df1]

> Membangun ide dari aplikasi yang kita miliki
> menggunakan sintaks yang bersih dantertata rapih
> tidak hanya menulis kode tetapi memahami permasalahnnya



## Tech

Dillinger uses a number of open source projects to work properly:

- [ReactJs] - Membangun Server Side Rendering dengan NextJs!
- [Prisma] - awesome web-based text editor
- [Zod] - Markdown parser done right. Fast and easy to extend.
- [PostgreSQL] - Menggunakan relasi atar table dengan FK
- [NextJs] - Untuk aplikasi server side rendering memuat SEO lebih baik
- [TailwindCSS] - Pustaka css yang baik powerful sederhana

And of course Dillinger itself is open source with a [public repository][dill]
 on GitHub.

## Installation

Dillinger requires [Node.js](https://nodejs.org/) v10+ to run.

Install the dependencies and devDependencies and start the server.

```sh
cd Aplikasi-Perhitungan-Asuransi
npm i
npm run dev
```

For production environments...

```sh
npm install --legacy-peer-deps
```

## Development

Want to contribute? Great!

Dillinger uses Gulp + Webpack for fast developing.
Make a change in your file and instantaneously see your updates!

Open your favorite Terminal and run these commands.

This will create the dillinger image and pull in the necessary dependencies.
Be sure to swap out `${package.json.version}` with the actual
version of Dillinger.

Once done, run the Docker image and map the port to whatever you wish on
your host. In this example, we simply map port 8000 of the host to
port 8080 of the Docker (or whatever port was exposed in the Dockerfile):

```sh
docker run -d -p 8000:8080 --restart=always --cap-add=SYS_ADMIN --name=dillinger <youruser>/dillinger:${package.json.version}
```

> Note: `--capt-add=SYS-ADMIN` is required for PDF rendering.

Verify the deployment by navigating to your server address in
your preferred browser.

```sh
127.0.0.1:8000
```
